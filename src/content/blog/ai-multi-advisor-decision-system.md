一位叫 Jaden 的创业者做了一件有点荒诞、但越用越觉得对的事：把六位他最尊敬的人，装进了他的工作系统。

德鲁克管业务，芒格看决策，乔布斯做产品，原研哉管架构，巴菲特帮拒绝，马斯克推执行。每次在对应领域要做决策，先问问他们。

这不是玄学，是一套实际在跑的 Claude Code 多顾问系统。

## 六位顾问各司其职

### 彼得·德鲁克 — 商务增长顾问

核心问题只有一个：「你的客户是谁？你在为他们创造什么价值？」

每次想推新功能、写新文章，先让德鲁克 Agent 问这两个问题。90% 的"好想法"在这里就死掉了——死得其所。实际效果：砍掉了 3 个看起来很好但没有用户需求的功能。

### 史蒂夫·乔布斯 — 产品设计顾问

「这是最好的体验吗？还是只是能用？」

做学生交付流程、内容视觉排版时让乔布斯过一遍。他会说：「这里多余，删掉。这里体验断了，重做。」产品页面砍掉一半内容，转化率反而上去了。

### 原研哉 — 系统架构顾问

「这个东西，有没有必要存在？」

少即是多不是口号，是设计哲学。工作区乱得像垃圾场时，让原研哉扫一遍：「这个文件夹的存在，是让系统更清晰，还是更混乱？」删掉 60%，剩下 40% 运转更顺。

### 查理·芒格 — 投资顾问

武器是多元思维模型：用物理学思考时间，用心理学思考人性，用经济学思考激励。

每次重大决策——招人、做某个市场、签某个合作——让芒格用至少 3 个不同学科框架来拆解。帮他识别了两次"感觉很好但实际是陷阱"的机会。

### 沃伦·巴菲特 — 商业顾问

只做一件事：帮你拒绝。

核心原则是护城河——你有什么别人抄不走的东西？每次焦虑"要不要跟上某个新趋势"，巴菲特通常会说：「专注你能做到最好的那件事，其他的无视。」

### 埃隆·马斯克 — 执行速度顾问

前五位帮你想清楚，马斯克帮你动起来。

「你为什么还没开始？完美是执行的敌人，先发出去，迭代比计划更有力量。」

还会用第一性原理拆解每一个"做不到"：「这是真正的限制，还是你自己设的边界？」被他问过之后，至少 3 件事从"打算做"变成了"已经发布"。

## 系统架构

这套系统对应的是一个三将军体系：

```
CEO（最终决策者）
│
├── 马斯克 Agent → 执行推动（直属）
│
├── 增长将军
│   ├── Jobs Agent → 产品体验
│   └── 原研哉 Agent → 系统极简
│
├── 商业将军
│   ├── 德鲁克 Agent → 价值定义
│   ├── 芒格 Agent → 多维决策
│   └── 巴菲特 Agent → 护城河
│
└── 交付将军
    └── 团队执行
```

每位顾问都有三个设计要素：
- 专属提问框架（最经典的思维方式）
- 专属决策领域（不越界，不重叠）
- 专属刹车机制（识别什么不该做）

决策流程：先匹配领域 → 调用对应顾问 → 自己拍板。

## 为什么这个思路值得关注

一个人的认知是有限的。但这六个人，每一个都花了几十年磨出了一套世界级的思维体系。

与其从零摸索，不如让他们的思维成为系统的一部分。这不是在抄他们，是在站在巨人的肩膀上做决策。

技术上，这套系统基于 Claude Code 的 Agent Team 功能，结合自定义 Skill 运行。不是装一下提示词就能跑的东西，需要长期磨合、持续迭代。

## 一个你现在就能用的方法

1. 选一个你最尊敬的人
2. 花 30 分钟，把他最核心的 3 个原则写下来
3. 装进你的系统提示词里
4. 下次做决策前，先让他过一遍

你会发现，决策质量会静悄悄地提升一个档次。

## 我的思考

这篇文章的价值不在于"用了哪六个人"，而在于提出了一种决策架构的思路：把不同领域的专家思维模式化、Agent 化，形成一个可复用的决策系统。

对于独立开发者和创业者来说，这种"认知外包"的方式特别有意义。你不需要真的请六个顾问，但你可以让 AI 扮演这些角色，用他们的框架来审视你的决策。

关键是：每个顾问要有明确的边界和专属的提问框架，不能让他们变成万能的"好好先生"。
